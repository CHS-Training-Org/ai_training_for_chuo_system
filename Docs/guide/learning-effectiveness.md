# 学習効果測定（満足度アンケート）

> 対象読者：メンター・リポジトリ管理者
> 参照：[operations-guide.md §質問サポートフロー](./operations-guide.md#support) / [curriculum.md](./curriculum.md) / [PHASE4_AI_DRIVEN_DEV_TASKS.md §5.4](../plan/PHASE4_AI_DRIVEN_DEV_TASKS.md)

本ページはアンケート様式と運用方針の真実の源です。  
定量指標（課題完了数・PR サイクルタイム等）の収集は本フェーズのスコープ外のため、ここでは満足度アンケートのみを扱います。

---

## アンケートの目的・運用 { #overview }

### 目的

- 学習者の体験（難易度・ツール有用性・ドキュメント品質）を定性的に把握し、チュートリアルの継続改善に役立てる。
- 改善提案を受け取る窓口を設け、学習者が主体的にフィードバックできる経路を確保する。

### 実施タイミング

| 実施タイミング | 推奨 |
|--------------|------|
| 必須ステップ（STEP-01〜05）を一通り完了したとき | ○ |
| 選択課題（エンハンス）を 1 件以上完了したとき | ○ |
| 学習期間の終了時 | ○ |

### 提出方法

メンターが「ふりかえり用 Issue」を起票し、学習者はその Issue にアンケート回答をコメントとして投稿してください。

!!! note "起票方法について"
    本リポジトリは `.github/ISSUE_TEMPLATE/config.yml` で自由記述 Issue を無効化しているため、学習者が Issue を直接起票することはできません。アンケート回収の際はメンターが事前に「ふりかえり用 Issue」を作成し、URL を学習者に共有してください。

### 集計・保管

メンターが各 Issue コメントを収集し、改善点を整理します。詳細は [§集計・活用](#review) を参照してください。

## アンケート設問 { #survey }

以下の設問について、評価と自由記述を記入して提出してください。  
評価欄は **1〜5 の数字**で回答します（1 = 低い／不満、5 = 高い／満足）。

### 評価設問

| # | 項目 | 評価（1〜5） | コメント（任意） |
|---|------|------------|----------------|
| 1 | **環境構築のしやすさ**：DevContainer の起動・各サービスの立ち上げまでスムーズに進められましたか？ | （記入） | （記入） |
| 2 | **難易度感**：課題全体の難易度はどう感じましたか？（1 = 易しすぎる、3 = ちょうどよい、5 = 難しすぎる） | （記入） | （記入） |
| 3 | **AI ツール（Claude Code）の有用性**：開発作業において Claude Code は役に立ちましたか？ | （記入） | （記入） |
| 4 | **ドキュメントの分かりやすさ**：ガイド・仕様書は理解しやすかったですか？ | （記入） | （記入） |
| 5 | **全体満足度**：このチュートリアルに総合的にどれくらい満足しましたか？ | （記入） | （記入） |

### 自由記述

| # | 質問 | 回答 |
|---|------|------|
| 6 | **良かった点**：特に役に立った・学びになったと感じた点を教えてください。 | （記入） |
| 7 | **改善してほしい点**：改善があると学習体験が向上すると思った点を教えてください。 | （記入） |

!!! note "記入のポイント"
    評価の数字だけでなく、**なぜそう感じたか**をコメント欄や自由記述に書いてもらえると、改善に活かしやすくなります。  
    「詰まった場所」「詰まらなかった理由」「AI ツールのどの使い方が効果的だったか」などの具体エピソードが特に参考になります。

## 集計・活用 { #review }

### 集計フロー

1. メンターが「ふりかえり用 Issue」のコメントを収集し、設問ごとに評価値と記述を整理する。
2. 改善提案を類型化し、優先度を判断する。
3. 採用する改善を新規 Issue としてタスク化する（[PHASE4_AI_DRIVEN_DEV_TASKS.md の更新ルール](../plan/PHASE4_AI_DRIVEN_DEV_TASKS.md) に従い該当カテゴリへ追記）。

### トリアージ方針

フィードバックの定期トリアージ（四半期など）で採否を判断します。改善提案の受付・トリアージ方針の詳細は [検討 B（メンテナンス方針）](../plan/PHASE4_AI_DRIVEN_DEV_TASKS.md) を参照してください。

質問・サポートフローとの棲み分けは [operations-guide.md §質問サポートフロー](./operations-guide.md#support) を参照してください。
